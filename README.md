## `$ whoami`

**KelvStream** is a premium, high-performance streaming platform that demonstrates the full client-to-server lifecycle of modern web video delivery. It features a live **HLS transcoding pipeline** powered by FFmpeg, adaptive bitrate switching, YouTube v3 API integration, and a bespoke luxury violet/obsidian UI вҖ” built entirely with React, TypeScript, and a custom CSS design system.

> *"Not a tutorial clone. A production-grade streaming architecture."*

---

## `$ ls features/`

### рҹҺ¬ Adaptive Bitrate Streaming (HLS)
The platform implements a real, local HLS transcoding pipeline вҖ” the same architecture used by Netflix, YouTube, and Twitch:

- **FFmpeg Transcoding** вҖ” uploaded `.mp4` files are processed by a custom Express backend that spawns an async FFmpeg child process
- **Dual-Profile Segmenting** вҖ” raw media is split into 6-second `.ts` chunks across two quality profiles:
  - `720p HD` вҖ” high quality, high bitrate
  - `360p SD` вҖ” standard quality, low bandwidth
- **Master Manifest** вҖ” a `master.m3u8` acts as the entry point for the ReactPlayer component to perform client-side **ABR shifts** in real time
- **Local HLS Badge** вҖ” uploaded videos surface at the top of the feed tagged `вҡЎ Local HLS`

### рҹҺЁ Premium Design System
- Deep obsidian body (`#0a0a0f`) with ultraviolet (`#7c3aed`) and pink neon (`#c084fc`) gradients
- Glassmorphism navbars and sidebars with `backdrop-filter: blur(20px)`
- Micro-animations on hover вҖ” play button overlays, card translations, pulsating pipeline loaders
- Fully responsive вҖ” flex + grid layouts for mobile and desktop

### рҹ“Ў YouTube v3 API Integration
Browse, search, and play remote YouTube content alongside local HLS streams вҖ” all in one unified feed.

## `$ cat INSTALL.md`

### Prerequisites

| Requirement | Version | Install |
|---|---|---|
| Node.js | v16+ | [nodejs.org](https://nodejs.org) |
| Yarn | latest | `npm install -g yarn` |
| FFmpeg | any | see below |

#### Install FFmpeg

```bash
# macOS
brew install ffmpeg

# Windows (winget)
winget install Gyan.FFmpeg
# then add FFmpeg /bin to your System PATH

# Ubuntu / Debian
sudo apt install ffmpeg
```

## `$ cat START.md`

### Step 1 вҖ” Backend (Transcoding Server)

```bash
cd server
npm install
npm start
```

> Server runs at `http://localhost:5000`
> Hosts the **Transcoding Control Panel** for raw video submission and pipeline monitoring.

---

### Step 2 вҖ” Frontend Client

```bash
cd client
yarn install
```

Create a `.env` file inside `client/`:

```env
VITE_REACT_APP_RAPID_API_KEY=your_rapidapi_key_here
```

> Get your free key at [rapidapi.com](https://rapidapi.com) вҶ’ search **YouTube v3**

```bash
yarn dev
```

> Client runs at `http://localhost:5173`
